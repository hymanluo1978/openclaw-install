OpenClaw 安装指南

从零开始，5-10 分钟完成 OpenClaw 的全新安装与配置。支持 Windows、macOS、Linux 全平台，含国内镜像加速方案。

macOS 详细教程

🍎 Mac Mini / MacBook 安装指南

适合新手的超详细分步安装教程（Apple Silicon M1/M2/M3/M4 通用）

1

打开 Terminal（终端）

按键盘上的 Command + 空格，弹出 Spotlight 搜索。输入 "Terminal"，然后按回车打开。Terminal 窗口会弹出来，这就是我们接下来所有操作的地方。

2

（推荐）先安装 Homebrew

虽然官方一键脚本会自动处理，但很多 Mac 用户反馈先装 Homebrew 可以避免一些权限或依赖问题。

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
🔑
执行过程中可能会让你输入 Mac 的登录密码（输入时屏幕不会显示字符，正常），输入完按回车。安装完后，关闭 Terminal 窗口，再重新打开一次（让环境变量生效）。

验证 Homebrew 安装成功：

brew --version
3

安装 Node.js（推荐用 Homebrew）

OpenClaw 需要 Node.js 22 或更高版本（官方推荐 Node 24）。

brew install node

安装完验证版本：

node --version
npm --version
💡
应该看到 v22.x.x 或 v24.x.x 以上。如果脚本自己装 Node 也可以，但提前装好更稳。
4

执行官方一键全新安装

直接复制下面这行命令，粘贴到 Terminal 后按回车：

curl -fsSL https://openclaw.ai/install.sh | bash
🔍 自动检测
检查并安装 Node.js（如果已装则跳过）
📥 全局安装
安装最新版 OpenClaw CLI
🚀 自动启动
自动启动 Onboarding（配置向导）

安装过程中可能会弹出几次权限请求（比如输入密码），正常输入即可。等待看到 "Installation finished successfully!" 或直接进入配置界面。

5

完成 Onboarding 配置向导

安装结束后，通常会自动进入配置向导。如果没有自动弹出，可以手动启动：

openclaw onboard
# 或快速向导
openclaw onboard --flow quickstart

向导里主要会让你设置：

🤖 AI 模型提供商
💬 聊天渠道
🌐 Gateway 后台服务
⚙️ 其他基础设置
📝
需要准备以下任意一个 API Key：OpenAI（ChatGPT）、Anthropic（Claude）、Grok（xAI）、通义千问（Qwen）等
6

验证安装成功

安装和配置完成后，验证 CLI 安装成功：

openclaw --version

启动 Gateway（后台服务）：

# 手动启动
openclaw gateway start

# 或用守护进程方式（开机自启推荐）
openclaw gateway install
7

启动并测试

如果你安装了 macOS App（部分用户会看到），可以直接打开 OpenClaw.app 来管理。否则，在浏览器访问本地 Gateway 地址（向导里通常会告诉你，比如 http://localhost:xxxx）。用你配置的聊天渠道（Telegram / Web）给你的 AI 发送第一条消息测试。

🛠️ Mac Mini 专属常见问题

如果提示需要 sudo，尝试在命令前加 sudo，或确保你的用户是管理员。

安装完后关闭 Terminal，重新打开一次。

运行 brew upgrade node 更新。

按 Control + C 中断，然后重新运行安装命令。

再次删除文件夹：rm -rf ~/.openclaw

如果一键脚本失败，可以手动安装：npm install -g openclaw@latest && openclaw onboard

Windows 安装

🪟 Windows 安装指南

原生支持 Windows，PowerShell 一键安装

1

准备工作（管理员权限）

右键点击开始菜单 → 选择 "Windows PowerShell (管理员)" 或 "终端 (管理员)"。

⚠️
重要：Windows 一定要用管理员 PowerShell 运行，否则可能报错。
2

执行全新安装命令

复制下面命令，直接粘贴并回车运行：

iwr -useb https://openclaw.ai/install.ps1 | iex
🔍 自动检测系统
智能识别 Windows 版本
📦 自动安装 Node.js
如果没有 Node.js 会帮你安装
🚀 自动启动向导
安装完成后自动进入配置
3

完成 Onboarding 配置

安装结束后，会自动进入配置向导（或手动运行 openclaw onboard）：

✅ 选择 QuickStart（快速开始）
✅ 配置 AI 模型 API Key
✅ 设置聊天渠道
✅ 配置 Gateway

整个过程通常 5-10 分钟就能完成。

4

验证与启动

安装完成后，运行以下命令检查：

openclaw --version

如果想手动启动 Gateway（后台服务）：

openclaw gateway start

或者用快速向导重新配置：

openclaw onboard --flow quickstart

📝 注意事项

⚠️
权限问题：Windows 一定要用管理员 PowerShell 运行，否则可能报错。
🗑️
旧残留：如果之前安装过，建议先删除 ~/.openclaw 文件夹再重新安装。
国内用户专属

🇨🇳 中国大陆安装方案

针对国内网络环境优化的安装方案，解决 GitHub 访问慢或失败的问题

🚀 推荐方案 1：国内镜像一键脚本（最快）

国内镜像安装 OpenClaw
已优化 npm 源、GitHub 加速，安装成功率高很多
curl -fsSL https://clawd.org.cn/install.sh | bash

🍺 推荐方案 2：Homebrew 国内镜像

在中国大陆安装 Homebrew 最常见的问题就是官方 GitHub 源访问慢或直接失败。以下是顺畅的安装方法：

中科大/清华/阿里镜像加速
自动检测系统（Intel/Apple Silicon）、选择国内镜像源、处理权限
/bin/zsh -c "$(curl -fsSL https://gitee.com/cunkai/HomebrewCN/raw/master/Homebrew.sh)"
💡
运行后会提示选择镜像源，输入 1（中科大源）或其他稳定选项，按回车即可。
极速精简版
更快但功能稍简，适合快速安装
/bin/zsh -c "$(curl -fsSL https://gitee.com/cunkai/HomebrewCN/raw/master/Homebrew.sh)" speed

使用第 1 种一键脚本安装 Homebrew，在中国大陆正常网络环境下，整体耗时一般在 2～8 分钟左右，大多数用户反馈是 3～5 分钟就能完成。

⚙️ 推荐方案 3：最稳妥的分步安装

适合有 Homebrew 且想完全掌控安装过程的用户：

1

确保 Homebrew 已安装并切换到国内源

使用之前用过的中科大/清华镜像。

2

安装 Node.js（必须 ≥22）

brew install node@22

然后把 Node 加到 PATH（Apple Silicon 通常是）：

echo 'export PATH="/opt/homebrew/opt/node@22/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
3

配置国内 npm 镜像

npm config set registry https://registry.npmmirror.com
4

安装 OpenClaw

npm install -g openclaw@latest
5

运行初始化向导

openclaw onboard

🔧 已有 Homebrew 配置国内镜像加速

如果已有 Homebrew，但更新/安装包很慢，配置国内镜像加速能让 brew install、brew update 速度提升 10 倍以上。

# 替换 brew 主仓库
git -C "$(brew --repo)" remote set-url origin https://mirrors.ustc.edu.cn/brew.git

# 替换 core 仓库（公式）
git -C "$(brew --repo homebrew/core)" remote set-url origin https://mirrors.ustc.edu.cn/homebrew-core.git

# （可选）cask 仓库
git -C "$(brew --repo homebrew/cask)" remote set-url origin https://mirrors.ustc.edu.cn/homebrew-cask.git

# 设置 Bottles（预编译二进制包）加速
echo 'export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles' >> ~/.zshrc
source ~/.zshrc
git -C "$(brew --repo)" remote set-url origin https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/brew.git

git -C "$(brew --repo homebrew/core)" remote set-url origin https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/homebrew-core.git

git -C "$(brew --repo homebrew/cask)" remote set-url origin https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/homebrew-cask.git

echo 'export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles' >> ~/.zshrc
echo 'export HOMEBREW_API_DOMAIN=https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles/api' >> ~/.zshrc
source ~/.zshrc

配置后运行 brew update 测试速度。以后每次新开终端记得 source ~/.zshrc（或重启终端）。

FAQ

❓ 常见问题

安装过程中可能遇到的问题及解决方案

官方命令 curl -fsSL https://openclaw.ai/install.sh | bash 在国内可能因网络问题失败。建议使用国内镜像：curl -fsSL https://clawd.org.cn/install.sh | bash，或采用分步安装方案。

安装完后关闭 Terminal/PowerShell，重新打开一次让环境变量生效。如果仍有问题，检查 Node.js 是否正确安装：node --version

删除配置目录：rm -rf ~/.openclaw（macOS/Linux）或删除对应的 Windows 配置文件夹。然后可以重新运行安装命令。

可以使用 npm 手动安装：

npm install -g openclaw@latest
openclaw onboard

支持以下任意一种：OpenAI（ChatGPT）、Anthropic（Claude）、Grok（xAI）、通义千问（Qwen）等。在 onboarding 向导中选择你的提供商并填入 API Key。

🎉 安装完成！

现在你可以通过配置的聊天渠道（Telegram / Web / WhatsApp 等）与你的 AI 助手对话了。
